## **壹、產品簡介**

**核心價值：資安防護的最後一哩路**

SOSI 是一套整合身分認證與遠端存取審計的高強度資安網關 。我們解決了企業在遠端維運時最頭痛的三大問題：「誰能進來？」、「能碰什麼？」以及「進來做了什麼？」 。



* 最強身分盾牌：強制整合 LDAP 認證與全站兩階段驗證（2FA），確保帳號不被盜用。
* 精確存取導航：透過管理員審核的授權機制，讓使用者只能連線至其職務所需的特定裝置 。
* 無死角行為鏡頭：所有的操作過程都會被錄影、鍵盤紀錄，並加上動態浮水印，讓非法操作無所遁形 。

![產品簡介](/sosi-manual/assets/images/review-1.png){: width="500px"}{: width="500px"}

## **貳、快速入門**


### **一、設定流程**

1. 建立使用者 



* **目的**：建立能登入 SOSI 系統的操作人員。
* **動作**：於「使用者列表」點擊「建立使用者」，設定其登入系統的帳號與密碼。

2. 建立裝置 



* **目的**：定義要連線的遠端主機（如 Windows 或 Linux）。
* **動作**：於「裝置列表」新增裝置，填入 **IP 地址**並選擇正確的**類型**（RDP、SSH 或 VNC）。

3. 建立憑證



* **目的**：儲存遠端主機的帳號密碼（這就是自動登入的「鑰匙」）。
* **動作**：於「憑證設定」建立憑證，輸入該主機真實有效的**帳號與密碼**。

4. 授權連線 



* **目的**：將「人、門、鑰匙」綁定在一起。
* **動作**：進入「授權裝置一覽」，點擊「建立裝置連線授權」，並在 **授權憑證 **欄位選取步驟 3 建立的憑證。


### **二、使用者連線**



1. 登入 SOSI 系統。
2. 在裝置列表找到該裝置卡片。
3. 點擊 「連線 」，系統將自動完成登入，直接進入桌面或指令介面。
4. 關閉網頁即可離開連線。




## **參、系統登入與帳號安全**

＊系統不開放註冊入口，只能結合 LDAP 目錄服務協定或是管理員手動新增使用者，讓使用者登入。


### **一、登入系統說明**



1. 使用者帳號：輸入使用者 ID 
2. 密碼：輸入登入密碼 
3. 認證方式：
* Admin 管理者：選擇 SOSI 
* 一般使用者：選擇 LDAP 

![登入畫面](/sosi-manual/assets/images/login-2.png){: width="500px"}

### **二、兩階段驗證 (2FA) 設定**

＊若有開啟則需設定



1. 登入後會顯示以下畫面
2. 綁定驗證器：使用手機下載 Authenticator App（如 Google Authenticator），掃描網頁顯示之 QR Code。

![兩階段驗證](/sosi-manual/assets/images/2FA-3.png){: width="500px"}

3. 輸入驗證碼：輸入 App 產生的 6 位數動態驗證碼完成綁定。(每30秒自動更換)
4. 備份恢復碼：系統會提供多組 恢復碼。

⚠️重要：請務必下載或截圖保存。當手機遺失或無法使用驗證器時，這是唯一的登入憑證（每組恢復碼僅能使用一次）。



5. 系統會跳回登入畫面，需重新輸入六位驗證碼並再次登入

![輸入驗證碼](/sosi-manual/assets/images/Verification-code-4.png){: width="500px"}

### **三、密碼錯誤鎖定與解除**



1.  帳號鎖定觸發條件
* 密碼錯誤次數：當使用者登入時，密碼輸入錯誤累計超過 5 次，系統將自動啟動鎖定機制 。
* 鎖定後果：該帳號會被系統自動鎖定，並暫停登入權限 1 小時 。

![密碼鎖定](/sosi-manual/assets/images/lock-5.png){: width="500px"}

2. 解除鎖定方式

    管理員可透過以下兩種方式協助使用者恢復帳號權限：

* 管理員手動解鎖：
    1. 進入「使用者列表」頁面 。
    2. 找到該受鎖定的使用者，點選解鎖按鈕 。
    3. 此動作會立即重置登入錯誤次數並恢復登入權限 。
* 系統自動解除：使用者只需等待 1 小時 的鎖定時間結束，系統便會自動解除限制 。
* 功能定義：解鎖功能僅用於重置登入錯誤次數，並不會更改原有的密碼 。

![解除密碼鎖定](/sosi-manual/assets/images/unlock-6.png){: width="500px"}

#### **（五）裝置設定**



1. SFTP 檔案傳輸設定

本區塊用於控管本機與遠端裝置間的檔案異動權限 。



* 啟用 SFTP（Enable SFTP）：勾選後，使用者可以在該裝置上進行本機與遠端裝置的檔案傳輸。
* 連線參數配置：
    * SFTP主機名稱 (hostname)：指定管理 SFTP 服務的主機名稱或 IP 位址 。
    * SFTP 連接埠 (port)：SFTP 的 SSH 服務通訊埠 。
    * SFTP 主機金鑰 (host key)：填入 SFTP 公鑰 。
    * SFTP 根目錄 (root directory)：設定傳輸檔案的預設初始根目錄 。
    * SFTP 伺服器存活間隔 (server alive interval)：設定 SSH 遠端連線傳送封包的間隔時間，以維持連線存活 。
    * SFTP 驗證類型 (Type) ：
        * SFTP Password Certificate (密碼憑證)：須輸入對應的帳號與密碼 。
        * SFTP PrivateKey Certificate (私鑰憑證)：須輸入帳號、私鑰及私鑰密碼 。

![檔案傳輸設定](/sosi-manual/assets/images/Sftp-13.png){: width="500px"}

2. 圖形會話錄製 (a)

本區塊提供完整的操作行為稽核紀錄設定 。



* 啟動錄影功能：勾選後，系統將於使用者連線時自動啟動畫面錄製 。
* 錄影時不包含鼠標：勾選後，錄製結果將隱藏使用者操作的鼠標軌跡與點選動作 。
* 錄影時包含鍵盤側錄功能：勾選後，系統會同步記錄連線時輸入的文字資訊，並產生 .txt 格式的鍵盤紀錄檔供後續稽核 。
3. 禁用剪貼板存取 (b)

此功能用於防止敏感資料透過複製貼上流出或進入遠端裝置 。

![其他設定](/sosi-manual/assets/images/-14.png){: width="500px"}

* 隱藏複製功能：開啟後，使用者將無法把遠端裝置內的資料複製到本機設備 。
* 隱藏貼上功能：開啟後，使用者將無法把本機資料貼上至遠端裝置 。

* 人員與對象：
    * 使用者帳號：選取欲授權的使用者 。
    * 使用者名稱：系統會根據選取的帳號自動顯示對應名稱 。
    * 裝置名稱：選取目標連線裝置 。
* 連線效期管控：
    * 授權起始/結束時間：若非永久授權，需定義可以連線的開始與截止時間 。
    * 不限連線時間：啟用此選項將設定為「永久授權」，此時「授權結束時間」將轉為不可更改狀態且不顯示時間 。
* 身分驗證加強：
    * 授權憑證：可將此授權與已存在之憑證綁定 。若有綁定憑證，「限制連線登入帳號」將強制設為「不限制」 。
    * 限制連線登入帳號：提供以下三種彈性選項 ：
        * 不限制：在未設定憑證時，由連線者自行輸入本機帳號密碼。
        * 同使用者帳號：未設定憑證時，系統自動套用使用者的「SOSI 帳號」登入裝置 。
        * 特定使用者帳號：未設定憑證時，強制使用於此處輸入的特定帳號進行登入 。

### **五、浮水印設定**

浮水印功能是系統預防資料外流的核心防護機制 。當使用者執行遠端連線時，浮水印會即時顯示於連線畫面，若發生重要資料外洩（如翻拍），管理員可藉此查詢資料出處 。



1. 同步稽核：除了即時連線畫面外，在裝置連線產生的螢幕錄影檔案中，浮水印也會同步顯示在畫面上 。
2. 功能配置說明：
* 啟用浮水印：管理者可依需求決定是否開啟全系統浮水印功能 。
* 文字字型與大小：可自定義顯示資訊的文字字型（如 sans-serif）與文字大小 (px) 。
* 文字顏色與透明度：管理者可設定文字顯示顏色，並調整在畫面上的透明度 (%) 以平衡安全性與操作視線 。
* 圖片規格設定：支援設定浮水印圖片的寬度與高度 (px)，以及圖片之間的間距 (px) 。
* 浮水印圖片上傳：
    * 格式限制：僅支援上傳 PNG 格式檔案 。
    * 大小限制：單個檔案最大限制為 1MB 。


### **六、領域設定**


#### （一）領域列表 

管理員可檢視系統內所有的認證領域配置 。



* 列表資訊：包含領域 ID、名稱、類型、建立時間及排序 。
* 排序重要性：列表順序將直接影響登入頁面中，「認證方式」下拉選單的顯示先後順序 。
* 搜尋功能：支援「領域名稱搜尋」，輸入關鍵字即可快速找尋特定領域 。
* 操作行為：
    * 查看/編輯/刪除
    * 同步領域使用者
    * 綁定測試

![領域設定](/sosi-manual/assets/images/domain-28.png){: width="500px"}

#### **（二）新增領域**

在新增或編輯領域時，請根據以下欄位定義進行配置 ：



* 領域名稱：領域顯示的名稱 。
* 類型：目前有兩種 ：
    * LdapAuthenticateStrategy：LDAP 或 Microsoft Active Directory 適用。
    * DatabaseAuthenticateStrategy：SOSI 內建的使用者帳密認證 。
* 伺服器：LDAP 伺服器的 IP 或 Hostname 。
* 連接埠：連線的 Port Number 。
* Base DN：類型為 LDAP 伺服器時使用的 Base DN 字串 。
* Bind DN：類型為 LDAP 伺服器時使用的 Bind DN 字串 。
* 使用者帳號屬性名稱：連線 LDAP 伺服器認證時使用的 LDAP 帳號屬性名稱，此屬性將會在認證通過時設定為使用者在 SOSI 系統中的使用者帳號 。
* 使用者顯示名屬性名稱：使用者顯示名稱的欄位 。
* 電子郵件屬性名稱：使用者 E-Mail 的欄位 。
* 驗證方式：目前只有「simple」與「anonymous」可選 。
* TLS 驗證方式：有四種選項，需依照該 LDAP 伺服器可用的方式設定 ：
    * VERIFY_NONE
    * VERIFY_PEER
    * VERIFY_FAIL_IF_NO_PEER_CERT
    * VERIFY_CLIENT_ONCE
* 加密方法：包含「無」、「simple_tls」與「start_tls」 。
* SSL 證書：如上一欄位不是無，則可輸入 SSL 證書，此欄位會加密 。
* SSL 加密金鑰：如上一欄位不是無，則可輸入 SSL 金鑰，此欄位會加密 。
* 識別物件格式：有兩種選項，需依照該 LDAP 伺服器可用的方式設定 ：
    * Distinguished Name
    * User Principal Name
* 使用者過濾條件：在 LDAP 伺服器中驗證一般使用者的 filter string 。
* 管理員過濾條件：在 LDAP 伺服器中驗證管理員的過濾條件 。
* 同步用帳號：使用同步功能時的 LDAP 伺服器使用者 。
* 同步用帳號的密碼：使用同步功能時的 LDAP 伺服器使用者密碼，此欄位會加密 。

#### **（二）郵件設定**

郵件設定是 SOSI 系統發送自動化通知的基礎。管理員必須在此正確配置 SMTP 伺服器，方能啟用以下核心服務：

1. 核心功能情境



* 帳號救援：發送「忘記密碼」重設連結與「帳號鎖定」解鎖通知信。
* 稽核通知：當連線錄影檔完成轉檔與打包後，自動發信通知相關人員下載。
* 連線測試：提供「發送測試信」功能，即時驗證郵件伺服器連線是否正常。

2. 收信地址來源



* 本地帳號：由管理員於「使用者列表」手動輸入。
* LDAP/AD 帳號：依據「領域設定」中的電子郵件屬性，從企業目錄自動同步。




### **八、安全審計**


#### **（ㄧ）連線紀錄列表**

此介面提供系統內所有裝置的連線歷史紀錄



1.  搜尋條件區塊

管理員可透過上方過濾器快速定位特定連線：



* 關鍵字搜尋：支援輸入「裝置識別名稱」或「使用者帳號」進行過濾。
* 時間區間：可設定「連線時間 ≥」或「關閉時間 ≤」來篩選特定時段的紀錄。
* 下拉選單：支援透過特定狀態（如連線狀態、轉檔狀態）進行進階篩選。
2. 列表欄位說明

表格顯示每一筆連線的詳細背景資訊：



* ID：連線事件的唯一編號。
* 裝置 ID：顯示該連線所屬的裝置名稱與連線協定（如 SSH）。
* 使用者帳號：執行該次連線的人員帳號。
* IP 地址：使用者連線時的來源或目標 IP 位址。
* 連線時間 / 關閉時間：記錄連線建立與結束的精確時刻。
* 狀態：顯示該連線的即時現況（如：已結束、連線失敗、連線中）。
* 轉檔狀態與時間：顯示螢幕錄影檔案的處理進度（如：完成、等待中）以及轉檔所耗費的時間。
3. 操作行為 

針對已結束且轉檔完成的紀錄，管理員可執行以下稽核動作：



* 播放：線上直接瀏覽該次連線的螢幕錄影畫面。
* 影片下載：將連線畫面的 MP4 影片檔案下載至本機存查。
* 鍵盤側錄：下載使用者連線期間的所有按鍵輸入紀錄（.txt 格式）。
* 連線報告：生成並下載該次連線的綜合稽核報告。
* 下載連線紀錄：位於畫面右上角，點選後可將符合搜尋條件的紀錄整批匯出（支援 .xlsx 或 .csv 格式）。

![連線紀錄列表](/sosi-manual/assets/images/connect-list-31.png){: width="500px"}

#### **（二）檔案傳輸紀錄列表**

此列表詳細記錄了所有透過系統執行的檔案上傳與下載行為，確保每一筆資料異動皆可追溯。



1. 前置觸發條件

檔案傳輸紀錄的產生，首要條件是該遠端裝置必須先被允許進行檔案傳輸：



* 設定位置：於「裝置設定頁面」的「File transfer via SFTP」區塊。
* 啟用動作：管理員必須勾選「允許檔案傳輸」選項，並填寫正確的 SFTP 連線資訊（如 Hostname, Port, 根目錄與認證憑證）。
* 運行機制：只有在裝置層級啟用了此功能，使用者連線時系統才會建立 SFTP 通道，進而捕捉並記錄檔案傳輸行為。
2. 列表顯示資訊與搜尋

管理員可透過「檔案傳輸紀錄列表」介面檢閱所有歷史紀錄：



* 搜尋條件：
    * 關鍵字搜尋：支援透過「連線 ID」、「裝置識別名稱」或「使用者帳號」進行快速篩選。
    * 上傳時間區間：可設定「上傳時間 ≥」或「上傳時間 ≤」來精確定位特定時段的傳輸活動。

![檔案傳輸紀錄列表](/sosi-manual/assets/images/filelist-32.png){: width="500px"}




#### **（三）操作紀錄列表**

此模組用於追蹤系統內所有物件（如帳號、裝置、憑證）的異動歷史，是確保管理合規性的核心功能。



1. 系統管理操作紀錄 

當管理員或使用者對系統資料執行任何修改時，系統會自動留存紀錄：



* 觸發機制：
    * 資料變更觸發：當系統偵測到資料庫物件執行 新增 (Create)、更新 (Update) 或 刪除 (Delete) 時，將自動產生紀錄。
2. 列表顯示資訊與搜尋

根據操作紀錄介面，管理員可利用以下功能進行稽核：



* 搜尋條件：
    * 關鍵字與過濾：可針對「操作者」、「IP 地址」、「事件類型（如建立）」或「項目類型（如登入、登出）」進行精確篩選。
    * 更新時間區間：可設定時間範圍（更新時間 ≥ 或 ≤）以定位特定時段的異動。
* 列表欄位說明：
    * 操作者：執行該動作的人員帳號。
    * 事件：動作類型（例如：建立、更新）。
    * IP 地址：操作者當時所在的來源 IP 位置。
    * 更新時間：動作發生的精確日期與時刻。
    * 項目類型：受影響的系統功能塊（如：登入、使用者、裝置）。
    * 項目編號：受影響物件在資料庫中的唯一 ID。
