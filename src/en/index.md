## **Chapter 1 ‚Äî Product Overview**

**Core Value: The Last Line of Defense in Cybersecurity**

SOSI is a high-strength security gateway that integrates identity authentication and remote access auditing. It solves the three biggest pain points enterprises face during remote maintenance operations: "Who can access?", "What can they touch?", and "What did they do after getting in?"



* **Strongest Identity Shield**: Enforces integration with LDAP authentication and site-wide two-factor authentication (2FA) to prevent account compromise.
* **Precise Access Navigation**: Through an administrator-reviewed authorization mechanism, users can only connect to the specific devices required for their role.
* **All-Angle Behavior Recording**: All operations are recorded via session recording and keylogging, and a dynamic watermark is applied, making unauthorized operations traceable.

## **Chapter 2 ‚Äî Quick Start**


### **I. Setup Flow**

1. Create a User



* **Purpose**: Create operators who can log in to the SOSI system.
* **Action**: On the **User List** page, click **Create User** and set the login account and password.

2. Create a Device



* **Purpose**: Define the remote host to connect to (e.g., Windows or Linux).
* **Action**: On the **Device List** page, add a device, enter the **IP Address**, and select the correct **Type** (RDP, SSH, or VNC).

3. Create a Certificate



* **Purpose**: Store the account credentials for the remote host (the "key" to automatic login).
* **Action**: Under **Certificate Settings**, create a certificate and enter the valid **username and password** for that host.

4. Authorize the Connection



* **Purpose**: Bind the "user, device, and key" together.
* **Action**: Go to **Accessible Device List**, click **Create Accessible Device**, and select the certificate created in Step 3 in the **Certificate** field.


### **II. User Connection Steps**



1. Log in to the SOSI system.
2. Find the target device card in the Device List.
3. Click **Connect**. The system will complete the login automatically and take you directly to the desktop or command-line interface.
4. Close the browser tab to end the connection.

### **III. Account Lockout and Unlock**



1. Lockout Trigger Conditions
* Failed Password Attempts: If a user enters an incorrect password 5 or more times consecutively, the system will automatically lock the account.
* Lock Consequence: The account will be automatically locked by the system, suspending login access for **1 hour**.

![Password Lockout](/sosi-manual/assets/images/lock-5.png){: width="500px"}

2. How to Unlock

    Administrators can restore a user's account access via the following two methods:

* Manual Unlock by Administrator:
    1. Go to the **User List** page.
    2. Find the locked user and click the **Unlock** button.
    3. This action immediately resets the failed login count and restores login access.
* Automatic System Unlock: The user simply needs to wait for the 1-hour lockout period to expire, after which the system will automatically lift the restriction.
* Function Definition: The Unlock function only resets the failed login count; it does **not** change the existing password.

![Unlock Account](/sosi-manual/assets/images/unlock-6.png){: width="500px"}

### **V. Watermark Settings**

The watermark feature is the core data-leakage prevention mechanism of the system. When a user initiates a remote connection, the watermark is displayed in real-time on the connection screen. If important data is leaked (e.g., photographed), administrators can use it to trace the source.



1. Synchronized Auditing: In addition to the live connection screen, the watermark is also simultaneously displayed in the screen recording files generated by device connections.
2. Feature Configuration:
* **Enable Watermark**: Administrators can decide whether to enable the system-wide watermark feature as needed.
* **Watermark Font and Size**: Customize the font (e.g., sans-serif) and font size (px) of the displayed information.
* **Watermark Font Color and Opacity**: Administrators can set the text display color and adjust the transparency (%) on the screen to balance security with operational visibility.
* **Image Size Settings**: Supports setting the watermark image's width and height (px), as well as the margin between images (px).
* **Watermark Image Upload**:
    * **Format Restriction**: Only PNG format files are supported.
    * **Size Restriction**: Maximum file size is 1MB.

#### **(ii) Add a Domain**

When adding or editing a domain, configure the following fields:



* **Name**: The display name of the domain.
* **Type**: Currently two types are available:
    * **LdapAuthenticateStrategy**: For use with LDAP or Microsoft Active Directory.
    * **DatabaseAuthenticateStrategy**: SOSI's built-in username/password authentication.
* **Host**: The IP address or hostname of the LDAP server.
* **Port**: The Port Number for the connection.
* **Base DN**: The Base DN string used when the type is LDAP.
* **Bind DN**: The Bind DN string used when the type is LDAP.
* **Username Attribute**: The LDAP account attribute name used for authentication when connecting to the LDAP server. This attribute will be set as the user's account in SOSI upon successful authentication.
* **Display Name Attribute**: The field for the user's display name.
* **Mail Attribute**: The field for the user's email address.
* **Auth Mode**: Currently only **simple** and **anonymous** are available.
* **TLS Verify Mode**: Four options are available; configure according to what the LDAP server supports:
    * VERIFY_NONE
    * VERIFY_PEER
    * VERIFY_FAIL_IF_NO_PEER_CERT
    * VERIFY_CLIENT_ONCE
* **Encryption Method**: Includes **None**, **simple_tls**, and **start_tls**.
* **SSL Cert**: If the previous field is not "None", you can enter an SSL certificate here. This field is encrypted.
* **SSL Key**: If the previous field is not "None", you can enter an SSL key here. This field is encrypted.
* **User Identify Format**: Two options are available; configure according to what the LDAP server supports:
    * Distinguished Name
    * User Principal Name
* **User Filter**: The filter string used to authenticate general users on the LDAP server.
* **Admin User Filter**: The filter condition used to authenticate administrators on the LDAP server.
* **Username for Sync**: The LDAP server user account used when using the sync feature.
* **Password of the User for Sync**: The LDAP server user's password for the sync feature. This field is encrypted.
