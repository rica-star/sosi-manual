## **第一章　製品概要**

**コアバリュー：情報セキュリティ対策の最後の砦**

SOSI は、ID 認証とリモートアクセス監査を統合した高強度のセキュリティゲートウェイです。企業がリモートメンテナンスで抱える三大課題、「誰がアクセスできるか？」「何を操作できるか？」「アクセス後に何を行ったか？」を解決します。



* 最強の ID シールド：LDAP 認証とサイト全体の二段階認証（2FA）を強制統合し、アカウントの不正使用を防止します。
* 精確なアクセス管理：管理者による審査を経た認証メカニズムにより、ユーザーは職務上必要な特定のデバイスにのみ接続できます。
* 死角のない行動記録：すべての操作はセッション録画・キーログが取られ、動的ウォーターマークが付与されるため、不正操作の追跡が可能です。

![製品概要](/sosi-manual/assets/images/review-1.png){: width="500px"}{: width="500px"}

## **第二章　クイックスタート**


### **一、設定の流れ**

1. ユーザーの作成



* **目的**：SOSI システムにログインできる操作者を作成します。
* **操作**：「ユーザー一覧」で「新規作成」をクリックし、ログイン用のアカウントとパスワードを設定します。

2. デバイスの作成



* **目的**：接続先のリモートホスト（Windows や Linux など）を定義します。
* **操作**：「デバイス一覧」でデバイスを追加し、**IP アドレス**を入力して正しい**タイプ**（RDP、SSH、または VNC）を選択します。

3. 証明書の作成



* **目的**：リモートホストのアカウントとパスワードを保存します（自動ログインの「鍵」です）。
* **操作**：「認証情報設定」で証明書を作成し、そのホストの有効な**アカウントとパスワード**を入力します。

4. 接続の認証



* **目的**：「ユーザー・デバイス・鍵」を一つに紐付けます。
* **操作**：「アクセス許可一覧」に進み、「デバイスアクセス許可を作成」をクリックして、**認証証明書**フィールドで手順 3 で作成した証明書を選択します。


### **二、ユーザーの接続手順**



1. SOSI システムにログインします。
2. デバイス一覧で対象のデバイスカードを見つけます。
3. 「接続」をクリックすると、システムが自動的にログインを完了し、デスクトップまたはコマンドラインインターフェースに直接入ります。
4. ウェブページを閉じると接続が終了します。




## **第三章　システムログインとアカウントセキュリティ**

＊システムは登録用の入口を公開していません。LDAP ディレクトリサービスとの連携、または管理者による手動でのユーザー追加のみが可能です。


### **一、ログイン方法**



1. ユーザーアカウント：ユーザー ID を入力します
2. パスワード：ログインパスワードを入力します
3. 認証方式：
* Admin 管理者：SOSI を選択
* 一般ユーザー：LDAP を選択

![ログイン画面](/sosi-manual/assets/images/login-2.png){: width="500px"}

### **二、二段階認証（2FA）の設定**

＊有効になっている場合は設定が必要です



1. ログイン後に以下の画面が表示されます
2. 認証アプリの設定：スマートフォンに Authenticator アプリ（Google Authenticator など）をダウンロードし、ウェブページに表示された QR コードをスキャンします。

![二段階認証](/sosi-manual/assets/images/2FA-3.png){: width="500px"}

3. 認証コードの入力：アプリが生成した 6 桁のワンタイム認証コードを入力して設定を完了します。（30 秒ごとに自動更新されます）
4. バックアップリカバリーコード：システムから複数のリカバリーコードが提供されます。

⚠️重要：必ずダウンロードまたはスクリーンショットで保存してください。スマートフォンを紛失したり認証アプリが使用できない場合、これが唯一のログイン手段となります（各リカバリーコードは一回のみ使用可能です）。



5. ログイン画面に戻りますので、再度 6 桁の認証コードを入力してログインしてください

![認証コード入力](/sosi-manual/assets/images/Verification-code-4.png){: width="500px"}

### **三、パスワード誤入力によるロックと解除**



1. アカウントロックの発動条件
* パスワード誤入力回数：ユーザーのログイン時にパスワードの誤入力が累計 5 回を超えると、システムは自動的にロックを実行します。
* ロックの結果：該当アカウントはシステムにより自動的にロックされ、1 時間ログイン権限が停止されます。

![パスワードロック](/sosi-manual/assets/images/lock-5.png){: width="500px"}

2. ロック解除の方法

    管理者は以下の 2 つの方法でユーザーのアカウント権限を回復できます：

* 管理者による手動ロック解除：
    1. 「ユーザー一覧」ページに移動します。
    2. ロックされたユーザーを見つけ、「ロック解除」ボタンをクリックします。
    3. この操作により、ログインエラー回数が即座にリセットされ、ログイン権限が回復されます。
* システムによる自動解除：ユーザーはロック時間の 1 時間が経過するまで待つだけで、システムが自動的に制限を解除します。
* 機能の定義：ロック解除機能はログインエラー回数のリセットのみを行い、元のパスワードは変更されません。

![パスワードロック解除](/sosi-manual/assets/images/unlock-6.png){: width="500px"}



## **第四章　管理者コントロールパネル**


### **一、デバイス一覧**


#### （一）デバイス一覧

現在接続可能なデバイス情報を表示します。管理者はシステム内のすべてのデバイスカード情報を確認できます

各デバイスはカード形式で表示され、以下のコア情報が含まれます：



* デバイス識別名：システム内でのデバイスの表示名。
* デバイス ID：システムが自動的に割り当てる一意の識別コード。
* IP／ホスト名：デバイスのネットワークアドレスまたはホスト名。
* デバイスタイプ：SSH、RDP、または VNC プロトコルに対応。
* サーバーポート番号：そのプロトコルが使用する通信ポート。
* 接続状態：「接続可能」または「満席」と表示されます。
* 接続人数：現在そのデバイスに接続しているユーザー数。


#### （二）デバイスの追加



* デバイス一覧ページで「デバイスを作成」をクリックします。
* 指定フィールドにデバイス情報を入力し、マシンタイプに応じた項目（VNC/RDP/SSH で必要なパラメータが異なります）を記入します。

![デバイス一覧](/sosi-manual/assets/images/device-7.png){: width="500px"}

#### （三）デバイスの表示



1. デバイス情報

管理者は「表示」機能からデバイスの詳細情報ページに入り、そのリソースの設定とアクセス権限を確認できます。



* デバイス識別名：システム内のデバイスの名称を表示します。
* デバイス IP／ホスト名：デバイスのネットワークアドレスまたはホスト名。
* デバイスタイプ：SSH、RDP、または VNC を含みます。
* サーバーポート番号：その接続プロトコルが使用する通信ポート。
* 備考：そのデバイスに関する追加説明情報。
* 管理者メールアドレス：そのデバイスの管理担当者の連絡先メールアドレス。

記録フィールドの説明：



* ID：接続イベントの一意の識別番号。

* 接続時間／終了時間：接続の開始と終了の時刻を記録します。

* キーログダウンロード：その接続で生成されたキーボード入力記録（.txt ファイル）をダウンロードします。
* 接続レポート

* 接続中の接続に対して、管理者は強制切断ボタンをクリックしたり、接続を監視したりすることができます

    強制切断の前に確認メッセージが表示されます。「確認」をクリックすると強制切断が実行されます

#### **（五）デバイス設定**



1. SFTP ファイル転送設定

このセクションは、ローカルとリモートデバイス間のファイル変更権限を管理するためのものです。



* SFTP を有効化（Enable SFTP）：チェックを入れると、ユーザーはそのデバイスでローカルとリモートデバイス間のファイル転送が行えます。
* 接続パラメータの設定：
    * SFTP ホスト名 (hostname)：SFTP サービスを管理するホスト名または IP アドレスを指定します。
    * SFTP ポート (port)：SFTP の SSH サービスの通信ポート。
    * SFTP ホストキー (host key)：SFTP の公開鍵を入力します。
    * SFTP ルートディレクトリ (root directory)：ファイル転送のデフォルト初期ルートディレクトリを設定します。
    * SFTP サーバー接続維持間隔 (server alive interval)：SSH リモート接続のパケット送信間隔を設定し、接続を維持します。
    * SFTP 認証タイプ (Type)：
        * SFTP Password Certificate（パスワード証明書）：対応するアカウントとパスワードの入力が必要です。
        * SFTP PrivateKey Certificate（秘密鍵証明書）：アカウント、秘密鍵、および秘密鍵パスフレーズの入力が必要です。

![ファイル転送設定](/sosi-manual/assets/images/Sftp-13.png){: width="500px"}

2. グラフィカルセッション録画 (a)

このセクションは、完全な操作行動の監査記録設定を提供します。



* 録画機能を有効化：チェックを入れると、ユーザーが接続した際にシステムが自動的に画面録画を開始します。
* 録画時にマウスを含まない：チェックを入れると、録画結果からユーザーのマウスカーソルの動きとクリック操作が非表示になります。
* 録画時にキーログ機能を含める：チェックを入れると、接続中に入力されたテキスト情報が同期して記録され、後続の監査用に .txt 形式のキーログファイルが生成されます。
3. クリップボードアクセスを無効化 (b)

この機能は、コピー＆ペーストを通じた機密データのリモートデバイスへの流入・流出を防止します。

![その他の設定](/sosi-manual/assets/images/-14.png){: width="500px"}

* コピーを無効化：有効にすると、ユーザーはリモートデバイス内のデータをローカルデバイスにコピーできなくなります。
* ペーストを無効化：有効にすると、ユーザーはローカルデータをリモートデバイスに貼り付けできなくなります。

2. デバイスアクセス許可の追加・編集
* アクセス許可一覧で「デバイスアクセス許可を作成」ボタンをクリックするか、一覧ページの「操作」から編集アイコンをクリックします。

![デバイスアクセス許可の追加](/sosi-manual/assets/images/authadd-24.png){: width="500px"}

* 認証するユーザーアカウント、デバイス名、認証証明書などの設定情報を選択します
* 認証証明書がない場合、接続者は接続確立時にリモートホストのアカウントとパスワードを手動で入力する必要があります。

![証明書認証](/sosi-manual/assets/images/cert-auth-25.png){: width="500px"}

3. フィールド設定の説明

デバイスアクセス許可の作成または編集時に、以下のキー情報を設定する必要があります：

![デバイスアクセス許可](/sosi-manual/assets/images/authdevice-26.png){: width="500px"}


* ユーザーと対象：
    * ユーザーアカウント：認証するユーザーを選択します。
    * ユーザー表示名：選択したアカウントに対応する表示名がシステムにより自動表示されます。
    * デバイス名：接続先のデバイスを選択します。
* 接続有効期間の管理：
    * 認証開始／終了時間：永久認証でない場合、接続可能な開始と終了時間を定義する必要があります。
    * 接続時間無制限：このオプションを有効にすると「永久認証」に設定され、「認証終了時間」は変更不可の状態になり時間は表示されません。
* ID 認証の強化：
    * 認証証明書：この認証を既存の証明書に紐付けることができます。証明書が紐付けられている場合、「接続ログインアカウントの制限」は強制的に「無制限」に設定されます。
    * 接続ログインアカウントの制限：以下の 3 つの柔軟なオプションが提供されます：
        * 無制限：証明書が設定されていない場合、接続者が自分のローカルアカウントとパスワードを入力します。
        * ログインユーザー名と同じ：証明書が設定されていない場合、システムはユーザーの「SOSI アカウント」を自動的にデバイスログインに適用します。
        * 特定のユーザー名：証明書が設定されていない場合、ここに入力した特定のアカウントを使用してログインします。

![接続ログイン](/sosi-manual/assets/images/connect-login-27.png){: width="500px"}

### **五、ウォーターマーク設定**

ウォーターマーク機能は、データ漏洩を防止するシステムのコア保護メカニズムです。ユーザーがリモート接続を実行すると、ウォーターマークが接続画面にリアルタイムで表示され、重要なデータ漏洩（撮影など）が発生した場合、管理者がデータの出所を追跡できます。



1. 同期監査：リアルタイムの接続画面に加え、デバイス接続で生成された画面録画ファイルにもウォーターマークが同期して表示されます。
2. 機能設定の説明：
* ウォーターマークを有効化：管理者は必要に応じてシステム全体のウォーターマーク機能を有効にするかどうかを決定できます。
* テキストのフォントとサイズ：表示情報のフォント（sans-serif など）と文字サイズ（px）をカスタマイズできます。
* テキストの色と透明度：管理者はテキストの表示色を設定し、セキュリティと操作の視認性のバランスを取るために画面上の透明度（%）を調整できます。
* 画像サイズ設定：ウォーターマーク画像の幅と高さ（px）、および画像間の間隔（px）の設定に対応しています。
* ウォーターマーク画像のアップロード：
    * 形式制限：PNG 形式のファイルのみアップロードに対応しています。
    * サイズ制限：1 ファイルあたり最大 1MB まで。


### **六、ドメイン設定**


#### （一）ドメイン一覧

管理者はシステム内のすべての認証ドメイン設定を確認できます。



* 一覧情報：ドメイン ID、名称、タイプ、作成日時、並び順を含みます。
* 並び順の重要性：一覧の順序はログインページの「認証方式」ドロップダウンメニューの表示順に直接影響します。
* 検索機能：「ドメイン名検索」に対応しており、キーワードを入力することで特定のドメインを素早く見つけられます。
* 操作：
    * 表示／編集／削除
    * ドメインユーザーの同期
    * バインドテスト

![ドメイン設定](/sosi-manual/assets/images/domain-28.png){: width="500px"}

#### **（二）ドメインの追加**

ドメインの追加または編集時は、以下のフィールド定義に従って設定してください：



* ドメイン名：ドメインの表示名。
* タイプ：現在 2 種類があります：
    * LdapAuthenticateStrategy：LDAP または Microsoft Active Directory に適用します。
    * DatabaseAuthenticateStrategy：SOSI 内蔵のユーザーアカウント認証。
* サーバー：LDAP サーバーの IP または ホスト名。
* ポート：接続する Port Number。
* ベース DN：LDAP サーバータイプを使用する場合のベース DN 文字列。
* バインド DN：LDAP サーバータイプを使用する場合のバインド DN 文字列。
* ユーザー名属性：LDAP サーバーに接続して認証する際に使用する LDAP アカウント属性名。この属性は認証通過時に SOSI システムにおけるユーザーアカウントとして設定されます。
* 表示名属性：ユーザーの表示名フィールド。
* メール属性：ユーザーのメールアドレスフィールド。
* 認証モード：現在「simple」と「anonymous」のみ選択可能です。
* TLS 認証モード：4 つのオプションがあり、その LDAP サーバーで利用可能な方式に従って設定します：
    * VERIFY_NONE
    * VERIFY_PEER
    * VERIFY_FAIL_IF_NO_PEER_CERT
    * VERIFY_CLIENT_ONCE
* 暗号化方法：「なし」、「simple_tls」、「start_tls」を含みます。
* SSL 証明書：前のフィールドが「なし」でない場合、SSL 証明書を入力できます。このフィールドは暗号化されます。
* SSL キー：前のフィールドが「なし」でない場合、SSL キーを入力できます。このフィールドは暗号化されます。
* ユーザー識別形式：2 つのオプションがあり、その LDAP サーバーで利用可能な方式に従って設定します：
    * Distinguished Name
    * User Principal Name
* ユーザーフィルター：LDAP サーバーで一般ユーザーを認証するためのフィルター文字列。
* 管理者ユーザーフィルター：LDAP サーバーで管理者を認証するためのフィルター条件。
* 同期用ユーザー名：同期機能を使用する際の LDAP サーバーユーザー。
* 同期用ユーザーのパスワード：同期機能を使用する際の LDAP サーバーユーザーパスワード。このフィールドは暗号化されます。

![ドメインの追加](/sosi-manual/assets/images/add-domain-29.png){: width="500px"}

### **七、全体設定**

管理者が全体設定でサイト全体の二段階認証を強制有効にすると、すべてのユーザーはログイン時に二段階認証コードを入力する必要があります。

![全体設定](/sosi-manual/assets/images/allset-30.png){: width="500px"}


#### **（二）メール設定**

メール設定は、SOSI システムが自動通知を送信するための基盤です。管理者はここで SMTP サーバーを正しく設定することで、以下のコアサービスが有効になります：

1. コア機能のシナリオ



* アカウント救援：「パスワードを忘れた場合」のリセットリンクと「アカウントロック」解除通知メールを送信します。
* 監査通知：接続録画ファイルの変換とパッケージングが完了した際に、関係者に自動でダウンロード通知メールを送信します。
* 接続テスト：「テストメール送信」機能を提供し、メールサーバーの接続状態をリアルタイムで確認できます。

2. 受信アドレスの来源



* ローカルアカウント：管理者が「ユーザー一覧」で手動入力します。
* LDAP/AD アカウント：「ドメイン設定」のメール属性に基づき、企業ディレクトリから自動同期されます。



### **八、セキュリティ監査**


#### **（一）接続記録一覧**

このインターフェースはシステム内のすべてのデバイスの接続履歴記録を提供します



1. 検索条件セクション

管理者は上部のフィルターで特定の接続をすばやく特定できます：



* キーワード検索：「デバイス識別名」または「ユーザーアカウント」の入力によるフィルタリングに対応しています。
* 時間範囲：「接続時間 ≥」または「終了時間 ≤」を設定して特定期間の記録を絞り込めます。
* ドロップダウンメニュー：特定の状態（接続状態、転送状態など）による高度なフィルタリングに対応しています。
2. 一覧フィールドの説明

テーブルには各接続の詳細な背景情報が表示されます：



* ID：接続イベントの一意の番号。
* デバイス ID：その接続が属するデバイス名と接続プロトコル（SSH など）が表示されます。
* ユーザーアカウント：その接続を実行した担当者のアカウント。
* IP アドレス：ユーザー接続時の送信元または宛先 IP アドレス。
* 接続時間 / 終了時間：接続の確立と終了の正確な時刻を記録します。
* 状態：その接続のリアルタイム現況（例：終了、接続失敗、接続中）を表示します。
* 転送状態と時間：画面録画ファイルの処理進捗（例：完了、待機中）と転送にかかった時間を表示します。
3. 操作

終了済みかつ転送完了の記録に対して、管理者は以下の監査アクションを実行できます：



* 再生：その接続の画面録画をオンラインで直接閲覧します。
* 動画ダウンロード：接続画面の MP4 動画ファイルをローカルに保管用としてダウンロードします。
* キーログ：ユーザーの接続中のすべてのキー入力記録（.txt 形式）をダウンロードします。
* 接続レポート：その接続の総合的な監査レポートを生成してダウンロードします。
* 接続記録をダウンロード：画面右上にあり、クリックすると検索条件に合致する記録を一括エクスポートできます（.xlsx または .csv 形式に対応）。

![接続記録一覧](/sosi-manual/assets/images/connect-list-31.png){: width="500px"}

#### **（二）ファイル転送記録一覧**

この一覧は、システムを通じて実行されたすべてのファイルアップロードおよびダウンロードの操作を詳細に記録し、すべてのデータ変更を追跡可能にします。



1. 事前トリガー条件

ファイル転送記録が生成されるための第一条件は、そのリモートデバイスがあらかじめファイル転送を許可されていることです：



* 設定場所：「デバイス設定ページ」の「SFTP 経由のファイル転送」セクション。
* 有効化アクション：管理者は「ファイル転送を許可」オプションにチェックを入れ、正しい SFTP 接続情報（ホスト名、ポート、ルートディレクトリ、認証証明書など）を記入する必要があります。
* 動作メカニズム：デバイスレベルでこの機能が有効になっている場合のみ、ユーザーの接続時にシステムが SFTP チャンネルを確立し、ファイル転送の動作をキャプチャして記録します。
2. 一覧表示情報と検索

管理者は「ファイル転送記録一覧」インターフェースからすべての履歴記録を確認できます：



* 検索条件：
    * キーワード検索：「接続 ID」、「デバイス識別名」、または「ユーザーアカウント」による素早い絞り込みに対応しています。
    * アップロード時間範囲：「アップロード日時 ≥」または「アップロード日時 ≤」を設定して特定期間の転送活動を正確に特定できます。

![ファイル転送記録一覧](/sosi-manual/assets/images/filelist-32.png){: width="500px"}


#### **（三）操作ログ一覧**

このモジュールはシステム内のすべてのオブジェクト（アカウント、デバイス、証明書など）の変更履歴を追跡し、管理のコンプライアンスを確保するためのコア機能です。



1. システム管理操作ログ

管理者またはユーザーがシステムデータに変更を加えると、システムは自動的に記録を保存します：



* トリガーメカニズム：
    * データ変更トリガー：システムがデータベースオブジェクトの 新規作成 (Create)、更新 (Update)、または 削除 (Delete) を検知すると、自動的に記録が生成されます。
2. 一覧表示情報と検索

操作ログインターフェースに基づき、管理者は以下の機能を使用して監査を行えます：



* 検索条件：
    * キーワードとフィルター：「操作者」、「IP アドレス」、「イベントタイプ（作成など）」、または「項目タイプ（ログイン、ログアウトなど）」による精確なフィルタリングができます。
    * 更新時間範囲：時間範囲（更新日時 ≥ または ≤）を設定して特定期間の変更を特定できます。
* 一覧フィールドの説明：
    * 操作者：その操作を実行した担当者のアカウント。
    * イベント：アクションタイプ（例：作成、更新）。
    * IP アドレス：操作者のその時点での送信元 IP アドレス。
    * 更新日時：操作が発生した正確な日時。
    * 項目タイプ：影響を受けたシステム機能ブロック（例：ログイン、ユーザー、デバイス）。
    * 項目 ID：影響を受けたオブジェクトのデータベース内の一意の ID。
